# 演習5：詳細設計・実装

## 1. 目的

演習4で作成したパッケージ図と設計クラス図をもとに、HRSを実際に操作できるプログラムとして実装する。

本演習では、予約、チェックイン、チェックアウトの一連の流れを、ブラウザ上で確認できるWebアプリケーションとして実装した。

## 2. 実装対象のユースケース

本実装では、以下の3つのユースケースを対象とした。

1. 空室を確認して部屋を予約する
2. 予約番号でチェックインする
3. 宿泊料を精算してチェックアウトする

加えて、デモ時に状態を確認しやすくするため、予約照会とデータ初期化も用意した。

## 3. 実装プログラム

実装プログラム： [HRS Grand Hotel Web UI](../implementation/HRS_grand_hotel_luxury_ui)

実行方法は以下の通りである。

```powershell
cd implementation\HRS_grand_hotel_luxury_ui
.\run.bat
```

または、以下のファイルをブラウザで直接開く。

```text
implementation/HRS_grand_hotel_luxury_ui/hotel_homepage/index.html
```

## 4. 設計クラス図との対応

演習4の設計クラス図を、以下のように実装へ対応させた。

| 設計上の役割 | 実装ファイル | 内容 |
|---|---|---|
| ユーザインタフェース層 | index.html, css/style.css, js/presentation.js | 画面表示、入力フォーム、ボタン操作、結果表示 |
| アプリケーション層 | js/application.js | 予約、チェックイン、チェックアウトの処理制御 |
| ドメイン層 | js/domain.js | Customer、Room、Reservation、Stay、Paymentなどのクラス |
| データソース相当 | js/datasource.js | localStorageを用いたデータ保存・読み出し |

## 5. 主なクラス・処理

### 5.1 presentation層

`presentation.js` では、利用者が画面で入力した内容を受け取り、application層の処理を呼び出す。

主な処理は以下である。

- 宿泊予約フォームの入力取得
- 客室一覧の表示
- 予約番号の表示
- チェックイン結果の表示
- チェックアウト時の宿泊料表示
- スクロール時の表示演出

### 5.2 application層

`application.js` では、ユースケースごとの処理を実装した。

| 処理 | 内容 |
|---|---|
| ReserveRoomControl | 空室検索、予約作成、予約番号発行 |
| CheckInRoomControl | 予約番号確認、部屋割り当て、宿泊開始処理 |
| CheckOutRoomControl | 宿泊情報確認、宿泊料計算、支払い記録、部屋解放 |
| ReservationManager | 予約の登録、検索、状態更新 |
| RoomManager | 部屋の検索、割り当て、状態更新 |
| PaymentManager | 宿泊料計算、支払い情報作成 |

### 5.3 domain層

`domain.js` では、HRSで扱うデータをクラスとして実装した。

- Customer
- HotelClerk
- Room
- Reservation
- Stay
- AccommodationFee
- Payment

### 5.4 datasource層

`datasource.js` では、ブラウザのlocalStorageを利用して、予約情報、部屋情報、宿泊情報、支払い情報を保存する。

これにより、DBサーバーを起動しなくても、ブラウザだけで予約からチェックアウトまでの動作を確認できる。

## 6. UIの内容

画面は、HRS Grand Hotel の公式サイトとして操作できるようにした。

主な画面要素は以下である。

- トップページ
- 客室一覧
- 宿泊予約フォーム
- チェックインフォーム
- チェックアウトフォーム
- 予約照会
- アクセス情報

所在地は「東京都新宿区大久保 3-4-1」とした。
また、トップページにはホテル外観画像、客室一覧にはスタンダード、デラックス、スイートの客室画像を表示した。

## 7. 動作確認

以下の流れで動作を確認した。

1. 宿泊日、宿泊人数、客室種別、姓、名、連絡先を入力して予約する
2. 予約番号が表示される
3. 予約番号を入力してチェックインする
4. 部屋番号が割り当てられる
5. 部屋番号を入力してチェックアウトする
6. 宿泊料が計算され、支払い済みとして記録される

## 8. 工夫点

演習4の設計で分けた役割を、実装でもできるだけ保つようにした。

画面の見た目だけを作るのではなく、予約、チェックイン、チェックアウトの業務処理が実際に通るようにした。
また、デモで見せやすいように、トップページ、客室画像、アクセス、スクロール時の表示演出を追加した。

当初はJava実装を想定していたが、デモで安定して操作できることを優先し、ブラウザだけで動作するHTML/CSS/JavaScript構成とした。
ただし、設計上の層分けとクラス責務は、演習4のパッケージ図・クラス図に対応するように維持した。
